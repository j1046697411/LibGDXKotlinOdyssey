## 章程检查

*门控: 必须在阶段 0 研究前通过. 阶段 1 设计后重新检查. *

### 核心原则检查
- [x] **卓越代码质量**: 代码将遵循 Kotlin 语言最佳实践和项目编码规范，具有清晰的意图和适当的注释
- [x] **严格测试标准**: 所有功能都将有相应的测试用例，测试覆盖率达到 80% 以上
- [x] **用户体验一致性**: 纯文字描述界面将保持一致的设计规范和交互模式
- [x] **高性能要求**: 代码将经过性能优化，确保游戏流畅运行
- [x] **模块化设计**: 每个功能将封装在独立的模块中，通过 ECS Addon 系统实现，避免循环依赖

### 治理文件

详细的项目原则和规则已记录在 [`constitution.md`](constitution.md) 中，包括：
- 代码质量标准 (风格、命名、注释、模块化)
- 测试要求 (覆盖率、单元/集成/性能测试)
- UX 一致性指南 (界面设计、交互流程、信息层级)
- 性能目标和优化规则
- 工作流程与质量门槛
