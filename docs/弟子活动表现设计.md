# 弟子活动表现设计

## 一、设计目标

设计一套直观、清晰的系统，让玩家能够快速了解弟子当前的活动状态、进度和预计完成时间，便于玩家管理和决策。

## 二、表现层次

### 5. 动态状态变化

**设计说明**：弟子活动状态应实时更新，并通过视觉效果反馈变化。

- **活动切换**：使用平滑过渡动画
- **效率变化**：效率提升时显示绿色向上箭头，下降时显示红色向下箭头
- **进度更新**：进度条实时变化，每5分钟更新一次
- **完成提示**：活动完成时显示庆祝动画和通知

### 8. 活动效率显示

**设计说明**：显示当前活动的效率百分比，帮助玩家了解弟子的表现。

- **效率>100%**：绿色显示，说明效率较高
- **效率=100%**：白色显示，正常效率
- **效率<100%**：黄色显示，效率较低
- **效率<80%**：红色显示，效率低下，需要关注

## 三、交互设计

### 1. 活动控制

- **暂停/继续**：允许玩家暂停或继续弟子的活动
- **加速/减速**：消耗资源加速活动进度
- **切换活动**：允许玩家直接为弟子分配新的活动
- **自动模式**：弟子根据优先级自动选择活动

### 2. 活动预设

- **修炼专注型**：优先修炼，次要任务
- **任务导向型**：优先完成任务
- **全面发展型**：均衡分配各项活动
- **工作效率型**：优先从事生产工作

## 四、实现建议

1. **数据驱动**：使用ECS组件存储弟子活动状态、进度、效率等数据
2. **实时更新**：每30秒更新一次活动进度
3. **事件触发**：关键节点触发事件通知
4. **视觉反馈**：使用动画和颜色变化增强反馈
5. **历史记录**：保存弟子最近的活动记录，便于追溯
6. **批量操作**：支持对多个弟子进行批量活动分配

## 五、设计优势

1. **直观清晰**：玩家能够快速了解弟子状态
2. **信息完整**：包含进度、效率、预期收益等关键信息
3. **交互便捷**：支持直接控制和调整弟子活动
4. **视觉统一**：使用统一的图标和色彩系统
5. **可扩展性**：便于添加新的活动类型
6. **个性化**：支持根据玩家喜好调整显示方式

## 六、应用场景

1. **弟子管理界面**：快速浏览所有弟子状态
2. **弟子详情界面**：查看单个弟子的详细活动信息
3. **宗门总览界面**：了解整体活动分布
4. **事件日志界面**：查看弟子活动历史
5. **通知系统**：实时接收重要活动通知

通过这套设计，玩家能够全面、直观地了解弟子正在做什么，便于做出合理的管理决策，提升游戏的管理体验和沉浸感。
