# 主界面框架设计（1920×1080优化版）

## 一、设计原则

1. **1920×1080优化**：针对160字符×60行分辨率优化布局
2. **视觉焦点比例**：左:中:右 = 2:5:2
3. **清晰层次**：通过边框、分隔线和缩进区分不同功能区域
4. **信息模块化**：将相关信息分组展示，便于用户快速获取
5. **交互友好**：提供明确的操作选项和状态反馈
6. **视觉平衡**：合理分配界面空间，避免信息拥挤
7. **快捷键支持**：提供常用操作的快捷键，提升操作效率
8. **状态可视化**：使用图标和颜色（通过文本符号模拟）表示不同状态
9. **响应式设计**：动态布局调整、可折叠区域设计，适应不同屏幕尺寸
10. **聚焦核心**：简化界面，聚焦核心功能，提升用户体验

## 三、区域功能说明

### 1. 顶部导航栏

**位置**：界面顶部，3行×160字符
**功能**：显示核心游戏信息，提供快速访问和全局搜索
**设计要点**：
- 第一行：显示当前区域、天气、时间、游戏速度、全局搜索框
- 第二行：显示宗门关键数据（弟子总数、资源、设施、占领区域等）
- 第三行：显示功能分类导航
- 支持点击展开详情模式
- 集成全局搜索功能

### 2. 左侧菜单栏

**位置**：界面左侧，52行×30字符
**功能**：提供游戏主要功能的快速导航
**设计要点**：
- 分级菜单设计
- 核心管理：宗门、弟子、资源、设施
- 功能区域：地图、任务、功法
- 生产系统：炼丹、炼器
- 系统：社交、设置
- 使用数字快捷键1-0,S快速访问
- 当前选中项高亮显示

### 3. 中央内容区

**位置**：界面中央，34行×100字符
**功能**：展示当前选中功能的详细内容
**设计要点**：
- 占据界面最大空间，是核心交互区域
- 顶部包含功能标签页，支持快速切换
- 主体内容根据标签页选择动态变化
- 信息排版清晰，操作按钮便于点击
- 支持滚动显示长列表

### 4. 右侧信息区

**位置**：界面右侧，15行×30字符
**功能**：显示角色信息和快捷操作
**设计要点**：
- 显示当前角色的核心属性
- 提供快捷操作按钮
- 显示当前状态加成
- 支持折叠/展开功能
- 保持简洁，避免信息过载

### 5. 底部状态栏

**位置**：界面底部，5行×160字符
**功能**：显示系统信息和快捷键提示
**设计要点**：
- 第一行：显示当前模式、游戏速度、运行时间、版本号
- 第二行：显示快捷键提示
- 第三行：显示消息提示
- 第四行：显示系统状态
- 第五行：显示开发者信息
- 支持滚动显示更多消息

## 八、设计优势

1. **符合1920×1080分辨率**：针对160字符×60行优化
2. **视觉焦点比例合理**：左:中:右 = 2:5:2，聚焦核心内容
3. **信息密度优化**：合并重复区域，精简导航栏
4. **导航效率提升**：优化菜单层级，增加快捷键
5. **视觉层次强化**：统一图标系统，优化间距
6. **响应式设计**：支持折叠/展开，适应不同屏幕尺寸
7. **交互友好**：提供明确的操作选项和状态反馈
8. **易于扩展**：设计灵活，支持后续功能扩展
9. **聚焦核心**：简化界面，聚焦核心功能
10. **统一风格**：所有界面使用一致的设计语言

## 九、实现建议

1. **模块化开发**：将每个区域作为独立模块开发
2. **状态管理**：使用状态管理库管理界面状态
3. **响应式布局**：实现动态布局调整
4. **快捷键支持**：实现完整的快捷键系统
5. **性能优化**：优化长列表滚动性能
6. **测试**：进行多场景测试，确保布局在不同情况下正常显示
7. **用户反馈**：收集用户反馈，持续优化布局
