# 修仙宗门GUI首页设计

## 一、设计原则

1. **模块化结构**：将界面划分为独立的功能模块，便于后续扩展和维护
2. **信息分层**：核心信息突出显示，次要信息有序排列
3. **操作便捷**：常用功能一键可达，减少用户操作路径
4. **视觉统一**：采用一致的设计语言和视觉风格
5. **响应式设计**：适配不同屏幕尺寸，保持良好的显示效果

### 2. 各组件详细设计

#### 2.1 顶部标题区

**位置**：界面顶部
**功能**：显示游戏名称、宗门名称和主要导航菜单
**设计要点**：
- 左侧显示游戏名称和宗门名称
- 右侧显示主要功能导航菜单
- 菜单采用水平排列，使用图标+文字组合
- 选中项高亮显示

#### 2.2 中央内容区

**位置**：界面中间部分
**功能**：显示宗门核心信息、弟子状态、资源情况等
**设计要点**：
- 分为左右两栏布局
- 左栏显示宗门弟子总览、宗门事件动态和弟子活动状态
- 右栏显示宗门资源总览、宗门任务列表和游戏状态信息
- 各模块使用分隔线明确区分

##### 2.2.1 宗门弟子总览
**内容**：显示不同层级弟子数量
**设计要点**：
- 按核心弟子、内门弟子、外门弟子、杂役分类显示
- 使用数字清晰表示数量

##### 2.2.2 宗门事件动态
**内容**：显示宗门最新事件通知
**设计要点**：
- 使用🔔图标标识事件
- 显示事件简要描述
- 按时间倒序排列

##### 2.2.3 弟子活动状态
**内容**：显示核心弟子当前活动
**设计要点**：
- 显示弟子名称、修为境界和当前状态
- 使用动态图标表示不同活动状态
- 按弟子重要性排序

##### 2.2.4 宗门资源总览
**内容**：显示宗门主要资源数量
**设计要点**：
- 显示资源当前数量和上限
- 使用对应图标增强视觉效果
- 按资源重要性排序

##### 2.2.5 宗门任务列表
**内容**：显示当前可接任务
**设计要点**：
- 使用不同图标区分任务类型（主线、日常、委托）
- 显示任务名称
- 按任务优先级排序

##### 2.2.6 游戏状态信息
**内容**：显示当前时间、游戏速度、自动存档状态和待处理事件数量
**设计要点**：
- 简洁明了，信息清晰
- 使用图标增强可读性

#### 2.3 底部功能区

**位置**：界面底部
**功能**：提供搜索功能和快速操作按钮
**设计要点**：
- 左侧为搜索框
- 右侧为快速操作按钮（快速行动、记笔记、设置）
- 按钮使用图标+文字组合

## 三、交互设计规范

### 1. 菜单交互
- 点击导航菜单可切换到对应功能页面
- 菜单支持鼠标悬停效果
- 当前页面在菜单中高亮显示

### 2. 信息交互
- 点击弟子名称可查看详细信息
- 点击任务可查看任务详情
- 点击资源可查看资源详情
- 点击事件可查看事件详情并处理

### 3. 快捷操作
- 搜索框支持快速搜索弟子、资源、任务等
- 快速行动按钮可快速执行常用操作
- 记笔记按钮可记录游戏心得
- 设置按钮可进入系统设置

### 4. 状态反馈
- 按钮点击有明显的视觉反馈
- 状态变化时有动态提示
- 重要事件有闪烁提醒

### 5. 响应式设计
- 支持不同屏幕尺寸自适应
- 小屏幕下可折叠部分信息
- 支持键盘快捷键操作

## 四、视觉设计规范

### 1. 色彩方案
- 主色调：青色（#4ECDC4）
- 辅助色：金色（#FFD93D）、红色（#FF6B6B）
- 背景色：深色（#2C3E50）
- 文字色：白色（#FFFFFF）

### 2. 字体规范
- 标题字体：微软雅黑，16px，粗体
- 正文字体：微软雅黑，14px
- 辅助文字：微软雅黑，12px，灰色

### 3. 图标规范
- 使用统一风格的emoji图标
- 图标与文字对齐
- 图标尺寸保持一致

### 4. 间距规范
- 模块间距：10px
- 行间距：1.5倍
- 内边距：8px

## 五、页面适配方案

### 1. 掌门模式
- 显示完整的宗门管理功能
- 突出显示宗门资源和弟子管理

### 2. 长老模式
- 聚焦于分管领域
- 显示相关领域的详细信息

### 3. 弟子模式
- 以个人修炼和任务为主
- 显示个人状态和可接任务

### 4. 观察者模式
- 显示全局信息
- 突出显示宗门发展和世界动态
